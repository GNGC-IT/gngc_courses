**Focus:** Programming logic, sensor calibration, and control loops.

- **Workshop Sessions:**
    
    - **Non-Blocking Code:** Moving students away from `delay()` towards `millis()` or interrupt-driven logic to ensure the robot remains responsive.
        
    - **Sensor Calibration:** Normalising data from infrared, ultrasonic, or encoders. Students should "analyse" the sensor noise and implement basic software filtering.
        
    - **Modular Programming:** Refactoring code into functions (e.g., `moveRobot()`, `readSensors()`) to meet the "High/Excellent" logic criteria.
        
- **Documentation Requirement:** * A flow-chart of the main program logic.
    
    - Code snippets showing how a specific technical hurdle was overcome.
